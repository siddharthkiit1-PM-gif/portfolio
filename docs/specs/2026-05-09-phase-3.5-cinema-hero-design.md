## 1. Goal

Replace the static post-Phase-3 hero with a **pinned, scroll-driven shader morph**: the volumetric orb morphs through liquid metal into a **chrome silhouette of the user's face**, with kinetic typography, depth-of-field, and post-process bloom. Apple-product-launch cinema. Mobile and reduced-motion ship a static SVG portrait fallback.

## 4. Shader morph (the heart of it)

Three SDF targets in `cinemaOrbShader.ts`, blended by a single `uMorph` uniform driven by ScrollTrigger:

```
sdfMix(p, uMorph) = mix(
  sphereSDF(p, 0.85),          // morph 0.0 вҶ’ 0.45
  capsuleSDF(p, 0.18, 1.4),     // morph 0.45 вҶ’ 0.7
  portraitSDF(p, uPortraitMask) // morph 0.7 вҶ’ 1.0
)
```

`portraitSDF` reads the alpha mask texture and uses a 2D distance-field approximation projected onto the front face of an extruded plane. The texture is generated from the user's photo (see В§6).

**Palette warp** uses `uPaletteShift` to mix the cosine palette toward chrome: `mix(cosinePalette(t), vec3(0.85,0.87,0.9), uPaletteShift)`.

**Rim light** between 0.65 and 0.85 morph: `pow(1-dot(n,-rd), 2.5) * mix(originalPalette, white, fresnelMix)` вҖ” that's the visible parting flourish at the silhouette boundary.

## 5. Post-processing

`@react-three/postprocessing` (already a peer of R3F):
- **UnrealBloom** вҖ” threshold animates 0.7 вҶ’ 0.55 across the morph; intensity 0.6 вҶ’ 0.95
- **DepthOfField** вҖ” focusDistance pinned to orb depth; bokehScale ramps 0 вҶ’ 4 between 35вҖ“75% morph
- Both **disabled on mobile and reduced-motion**

## 6. Portrait alpha mask pipeline

Source asset committed: `public/assets/portrait-source.jpg` (the photo provided by the user, ~115KB).

**Build-time processing** (one-shot script, `scripts/build-portrait-mask.ts`):
1. Use `@imgly/background-removal-node` to alpha-cut the portrait
2. Output two assets:
   - `public/portrait/portrait-1024.png` вҖ” RGBA cutout, used for the static fallback
   - `public/portrait/portrait-mask-512.png` вҖ” single-channel alpha (white silhouette on black), used as the shader's `uPortraitMask` sampler
3. Mask is centered, head-and-shoulders crop with 10% margin

The script runs once locally (`pnpm portrait:build`), commits the outputs. Not wired into CI to avoid hot-loading the ML model on every Vercel build.

**Fallback if `@imgly/background-removal-node` proves heavy:** swap to Apple's Vision framework via a quick `sips` + manual alpha trace, OR use an online tool once. Acceptable result is a clean head + shoulders alpha with no jagged edges.

## 7. Typography choreography

- **GSAP SplitText** wraps the headline; characters become individual elements
- **0вҖ“25%**: each character translates toward the orb center with a stagger-based easing curve, fades to opacity 0.35, rotates Вұ18В°
- **50%**: a fresh kinetic line ("Built across AI, health, and consumer.") word-by-word, ~140ms per word, with a violet caret cursor dwell
- **75%**: name ("Siddharth Agrawal.") starts coalescing вҖ” characters re-emerge from the silhouette area, opacity 0 вҶ’ 0.7, slight rotational settle
- **100%**: full name 96px Inter Light/Regular, subline 18px, CTAs slide in from `x: -20px` with stagger 0.06s, status pill fades in

## 10. Reduced-motion + mobile fallback

- `useReducedMotion()` true OR `matchMedia("(max-width: 767px)").matches` true вҶ’
  - No canvas, no pin, no GSAP timeline, no Lenis hook from Phase 3
  - Render `<HeroOrbFallback />` which displays `portrait-1024.png` centered, with the bloom/DOF effects faked as a CSS radial-gradient halo behind it
  - All headline copy at full opacity, regular flow
- Same content parity (headline, subline, CTAs, status pill) вҖ” only the morph + DOF + bloom drop

## 11. What's explicitly NOT in this phase

- Cursor parallax driving silhouette rotation post-pin (deferred to Phase 5)
- Audio toggle / sound design (deferred)
- Different morph paths for case-study deep-links (deferred)
- Editing the portrait via admin mode (the source photo is committed to the repo for now; admin-edit upload of a new portrait is a Phase 6+ feature)
