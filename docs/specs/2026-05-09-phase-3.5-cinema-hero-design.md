## 1. Goal

Replace the static post-Phase-3 hero with a **pinned, scroll-driven shader morph**: the volumetric orb morphs through liquid metal into a **chrome silhouette of the user's face**, with kinetic typography, depth-of-field, and post-process bloom. Apple-product-launch cinema.

**Cinema everywhere, tuned per device.** The morph ships on phones, tablets, and desktop ‚Äî not desktop-only. Three quality tiers, plus a static fallback for low-end devices and reduced-motion. Layout adapts to portrait viewports.

## 4. Shader morph (the heart of it)

Three SDF targets in `cinemaOrbShader.ts`, blended by a single `uMorph` uniform driven by ScrollTrigger:

```
sdfMix(p, uMorph) = mix(
  sphereSDF(p, 0.85),          // morph 0.0 ‚Üí 0.45
  capsuleSDF(p, 0.18, 1.4),     // morph 0.45 ‚Üí 0.7
  portraitSDF(p, uPortraitMask) // morph 0.7 ‚Üí 1.0
)
```

`portraitSDF` reads the alpha mask texture and uses a 2D distance-field approximation projected onto the front face of an extruded plane. The texture is generated from the user's photo (see ¬ß6).

**Palette warp** uses `uPaletteShift` to mix the cosine palette toward chrome: `mix(cosinePalette(t), vec3(0.85,0.87,0.9), uPaletteShift)`.

**Rim light** between 0.65 and 0.85 morph: `pow(1-dot(n,-rd), 2.5) * mix(originalPalette, white, fresnelMix)` ‚Äî that's the visible parting flourish at the silhouette boundary.

## 5. Post-processing (tiered)

`@react-three/postprocessing`:

| Tier | UnrealBloom | DepthOfField | Notes |
|---|---|---|---|
| high (desktop) | threshold 0.7‚Üí0.55, intensity 0.6‚Üí0.95 | bokehScale 0‚Üí4 | Full pass |
| mid (tablet)   | threshold 0.7, intensity 0.55 (static) | off | Bloom only, no DOF |
| low (mobile)   | off | off | Raw shader; bloom is faked inside the fragment via emissive boost |
| static         | off | off | No canvas at all |

The shader exposes a `uBloomFake` uniform on `low` tier ‚Äî a small additive halo computed in-frag ‚Äî to keep the look consistent without a second pass.

## 6. Portrait alpha mask pipeline

Source asset committed: `public/assets/portrait-source.jpg` (the photo provided by the user, ~115KB).

**Build-time processing** (one-shot script, `scripts/build-portrait-mask.ts`):
1. Use `@imgly/background-removal-node` to alpha-cut the portrait
2. Output two assets:
   - `public/portrait/portrait-1024.png` ‚Äî RGBA cutout, used for the static fallback
   - `public/portrait/portrait-mask-512.png` ‚Äî single-channel alpha (white silhouette on black), used as the shader's `uPortraitMask` sampler
3. Mask is centered, head-and-shoulders crop with 10% margin

The script runs once locally (`pnpm portrait:build`), commits the outputs. Not wired into CI to avoid hot-loading the ML model on every Vercel build.

**Fallback if `@imgly/background-removal-node` proves heavy:** swap to Apple's Vision framework via a quick `sips` + manual alpha trace, OR use an online tool once. Acceptable result is a clean head + shoulders alpha with no jagged edges.

## 7. Typography choreography

- **GSAP SplitText** wraps the headline; characters become individual elements
- **0‚Äì25%**: each character translates toward the orb center with a stagger-based easing curve, fades to opacity 0.35, rotates ¬±18¬∞
- **50%**: a fresh kinetic line ("Built across AI, health, and consumer.") word-by-word, ~140ms per word, with a violet caret cursor dwell
- **75%**: name ("Siddharth Agrawal.") starts coalescing ‚Äî characters re-emerge from the silhouette area, opacity 0 ‚Üí 0.7, slight rotational settle
- **100%**: full name 96px Inter Light/Regular, subline 18px, CTAs slide in from `x: -20px` with stagger 0.06s, status pill fades in

**Touch interaction:**
- Vertical swipe drives the morph (ScrollTrigger pin handles this natively)
- `ScrollTrigger.normalizeScroll(true)` enabled on touch devices to smooth iOS Safari momentum/bounce
- Tap-and-drag inside the canvas does NOT rotate the silhouette in this phase (deferred); pointer events on the canvas use `pointer-events: none`
- `touch-action: pan-y` on the hero to allow vertical scroll, block horizontal gestures from hijacking
- iOS double-tap zoom prevented via `<meta viewport ... user-scalable=no>` already in the layout

**iOS Safari quirks handled:**
- Use `100dvh` everywhere (not `100vh`) so the address bar collapse doesn't jank the pin
- Lenis is **disabled on iOS** specifically (it conflicts with Safari's native momentum and ScrollTrigger.normalizeScroll); GSAP handles smoothing instead
- `transform: translateZ(0)` on the canvas wrapper to force GPU compositing
- `-webkit-backface-visibility: hidden` to prevent flicker during pin transition

**Android quirks handled:**
- Some Android browsers report `deviceMemory` 4 but throttle aggressively; the runtime FPS probe (¬ß9) catches this and downgrades.
- `requestAnimationFrame` is throttled in background tabs ‚Äî IntersectionObserver pause handles it.

## 10b. Static fallback (reduced-motion + low-end devices)

Triggered when `tier === "static"`:
- No canvas, no pin, no GSAP timeline, no smooth scroll
- Renders `<HeroOrbFallback />`: `portrait-1024.png` centered, with bloom/DOF faked as a CSS radial-gradient halo behind it (same chrome look)
- All headline copy at full opacity in regular flow
- Layout still adapts: desktop two-column / tablet stacked / phone stacked
- Same content parity with cinema variants

## 11. What's explicitly NOT in this phase

- Cursor parallax driving silhouette rotation post-pin (deferred to Phase 5)
- Audio toggle / sound design (deferred)
- Different morph paths for case-study deep-links (deferred)
- Editing the portrait via admin mode (the source photo is committed to the repo for now; admin-edit upload of a new portrait is a Phase 6+ feature)
